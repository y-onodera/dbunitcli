Dataset Settings

テーブルごとのメタデータ（主キー・カラム選択・ソート・フィルタなど）を定義する設定ファイルです。settings 配列で個別のテーブル設定を、commonSettings 配列で複数テーブル共通の設定を記述します。

settings のフィールド

フィールド説明
name対象テーブル名。スキーマを含める場合は schema.table 形式
keys主キーのカラム名（複数指定可）
include処理対象にするカラム
exclude除外するカラム
orderソート順に使うカラム
filterデータ選択条件（JExl 式）
distinct重複除去フラグ

commonSettings

settings と同じ形式で、複数テーブルに適用される共通設定を定義します。

上書きルール

  1. settings の設定が優先されます
  2. 配列（include、exclude など）は結合されます
  3. フラグ（distinct）は上書きされます

import

外部の設定ファイルを読み込んで再利用できます。

{
  "import": [
    { "path": "./common/base.json" },
    { "path": "/abs/path/conf.json" }
  ]
}

パス解決の優先順位

  1. 絶対パス
  2. 設定ファイルからの相対パス
  3. カレントディレクトリからの相対パス

記述例

{
  "settings": [
    {
      "name": "users",
      "keys": ["id"],
      "exclude": ["created_at", "updated_at"],
      "order": ["id"]
    }
  ],
  "commonSettings": [
    {
      "exclude": ["created_at", "updated_at"]
    }
  ]
}
