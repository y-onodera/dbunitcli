Dataset Settings

テーブルごとのメタデータ（主キー・カラム選択・ソート・フィルタなど）を定義する設定ファイルです。settings 配列で個別のテーブル設定を、commonSettings 配列で複数テーブル共通の設定を記述します。各フィールドの UI 操作・記述例や JeXL 式の書き方は Dataset Setting を参照してください。

settings のフィールド

各フィールドの詳細は Dataset Setting を参照してください。

Target（対象指定）

フィールド説明
name対象テーブル名。単一文字列・配列、または { any, filePath } 形式で外部ファイル参照も可。スキーマを含める場合は schema.table 形式
filePathname と併用し、テーブル名リストを読み込むファイルパス
pattern正規表現によるテーブル名マッチ。{ string, exclude } 形式で除外パターンの指定も可
innerJoin2 テーブルを内部結合して 1 つの論理テーブルとして扱う（left / right / column / on）
outerJoin2 テーブルを外部結合して 1 つの論理テーブルとして扱う（left / right / column / on）
fullJoin2 テーブルを完全外部結合して 1 つの論理テーブルとして扱う（left / right / column / on）

Output（テーブル名・分割）

フィールド説明
prefix出力テーブル名に付与する接頭辞
tableName出力テーブル名のベース名（リネーム）
suffix出力テーブル名に付与する接尾辞
split行分割設定。内部フィールド prefix / tableName / suffix / breakKey / filter / limit を持つ
separateネストしたサブ設定配列。1 つの入力から複数の派生テーブルを生成する

Columns（カラム選択）

フィールド説明
keys主キーのカラム名（複数指定可）
include処理対象にするカラム
exclude除外するカラム
orderソート順に使うカラム

Additional Columns（追加カラム）

フィールド説明
string文字列型の追加カラム。値は JeXL 式（カラム名をキーとするオブジェクト）
number数値型の追加カラム。値は JeXL 式（カラム名をキーとするオブジェクト）
boolean真偽値型の追加カラム。値は JeXL 式（カラム名をキーとするオブジェクト）
function関数表現による追加カラム。複雑なロジックを JeXL 関数として定義

Rows（行フィルタ）

フィールド説明
filterデータ選択条件の JeXL 式（複数指定時は AND 扱い）
distinct重複除去フラグ

commonSettings

settings と同じ形式で、複数テーブルに適用される共通設定を定義します。

上書きルール

  1. settings の設定が優先されます
  2. 配列（include、exclude など）は結合されます
  3. フラグ（distinct）は上書きされます

import

外部の設定ファイルを読み込んで再利用できます。

{
  "import": [
    { "path": "./common/base.json" },
    { "path": "/abs/path/conf.json" }
  ]
}

パス解決の優先順位

  1. 絶対パス
  2. 設定ファイルからの相対パス
  3. カレントディレクトリからの相対パス

記述例

{
  "settings": [
    {
      "name": "users",
      "keys": ["id"],
      "exclude": ["created_at", "updated_at"],
      "order": ["id"]
    }
  ],
  "commonSettings": [
    {
      "exclude": ["created_at", "updated_at"]
    }
  ]
}
