StringTemplate 4 書式リファレンス

DBUnit CLI のテンプレート処理では StringTemplate 4 (ST4) を使用します。テンプレート内の値は $値:書式名()$ の形式でフォーマットできます。ST4 標準の書式に加えて、DBUnit CLI 独自の拡張書式が利用可能です。

基本構文

書式は ST4 のレンダラー呼び出しとして、属性参照の直後に : で連結します。

$param.columnName:書式名()$

区切り文字が変更されている場合は、対応する文字で囲みます（例: <param.columnName:書式名()>）。詳細は Template の templateVarStart / templateVarStop を参照してください。

ST4 標準書式

ST4 の StringRenderer が提供する標準書式です。

書式名説明入力例出力例
upper大文字に変換helloHELLO
lower小文字に変換HELLOhello
cap先頭を大文字に変換hello worldHello world
url-encodeURL エンコードa ba+b
xml-encodeXML エンティティにエスケープ（& < 等）<tag>&lt;tag&gt;

DBUnit CLI 拡張書式（SqlEscapeStringRenderer）

DBUnit CLI は SqlEscapeStringRenderer により以下の拡張書式を追加しています。

命名規則変換

書式名説明入力例出力例
camelToSnakeキャメルケース → スネークケース（小文字）userIduser_id
camelToUpperSnakeキャメルケース → スネークケース（大文字）userIdUSER_ID
snakeToCamelスネークケース → キャメルケース（先頭小文字）user_iduserId
snakeToUpperCamelスネークケース → アッパーキャメルケース（先頭大文字）user_idUserId
camelToKebabキャメルケース → ケバブケース（小文字）userIduser-id
camelToUpperKebabキャメルケース → ケバブケース（大文字）userIdUSER-ID
kebabToCamelケバブケース → キャメルケース（先頭小文字）user-iduserId
kebabToUpperCamelケバブケース → アッパーキャメルケース（先頭大文字）user-idUserId

使用例（カラム名 userId を SQL の列定義に使う）:

ALTER TABLE $param.tableName$ ADD COLUMN $param.columnName:camelToSnake()$ VARCHAR(255);
-- 入力: tableName=userProfile, columnName=firstName
-- 出力: ALTER TABLE userProfile ADD COLUMN first_name VARCHAR(255);

SQL エスケープ

書式名説明
escapeSqlシングルクォートを '' にエスケープして SQL 文字列リテラルとして安全に出力する。4,000 バイト超の値は Oracle の TO_CLOB() に分割して連結。改行文字（\n / \r\n）は CHR(10) / CHR(13) || CHR(10) で連結

使用例:

INSERT INTO messages (body) VALUES ($param.body:escapeSql()$);
-- 入力: body = It's a test
-- 出力: INSERT INTO messages (body) VALUES ('It''s a test');

-- 入力: body = line1\nline2
-- 出力: INSERT INTO messages (body) VALUES ('line1' || CHR(10) || 'line2');

4,000 バイトを超える長文テキストの場合:

-- 出力例（4,001 バイトの文字列）:
-- TO_CLOB('...最初の4000バイト...') || TO_CLOB('...残りのバイト...')

式展開エスケープ

テンプレート内で別のテンプレートエンジンの式記法をそのまま出力したい場合に使います。

書式名説明入力例出力例
jexlExp値を JEXL 式 ${...} として出力user.name${user.name}
ST4Exp値を ST4 式 $...$ として出力（ST4 テンプレートを生成するテンプレートで使用）param.id$param.id$

使用例（ST4 テンプレートを生成する ST4 テンプレート）:

-- 入力: expr = param.tableName
-- テンプレート:
SELECT * FROM $param.expr:ST4Exp()$;

-- 出力（ST4 式が文字列としてそのまま埋め込まれる）:
SELECT * FROM $param.tableName$;

複合例

命名規則変換と SQL エスケープを組み合わせた例です。

テンプレート (insert.sql):
  INSERT INTO $param.tableName:camelToSnake()$
    ($param.col1:camelToSnake()$, $param.col2:camelToSnake()$)
  VALUES
    ($param.val1:escapeSql()$, $param.val2:escapeSql()$);

入力行:
  tableName = userProfile
  col1      = firstName
  col2      = lastName
  val1      = O'Brien
  val2      = Alice

レンダリング結果:
  INSERT INTO user_profile
    (first_name, last_name)
  VALUES
    ('O''Brien', 'Alice');

ST4 テンプレートの書き方

以下の構文は組み込みテンプレート（core/src/main/resources/ 以下の .txt / .stg）で実際に使われているパターンです。

変数参照

$属性名$ で値を展開します。. でプロパティをたどれます。

-- テンプレート (本体 .txt)
insert into $tableName$ values ($param.value$);

-- 入力: tableName=USERS, param.value=1
-- 出力:
insert into USERS values (1);

繰り返し（イテレーション）

$リスト:{変数 | 本文}$ でリストを繰り返し展開します。; separator="区切り文字" で要素間の区切りを指定できます。

組み込み INSERT テンプレート（sql/sqlTemplate.stg）での使用例：

-- .stg（グループファイル）内のサブテンプレート定義
insert() ::= <<
$rows:{row |insert into $tableName$ ($columns:{column | $column.columnName$ }; separator=","$) values (...);
}$
>>

-- 入力: tableName=USERS, columns=[id, name, age], rows=[行1, 行2]
-- 出力:
insert into USERS (id,name,age) values (...);
insert into USERS (id,name,age) values (...);

条件分岐

$if(条件)$...$elseif(条件)$...$else$...$endif$ で条件によって出力を切り替えます。! で否定、strip() でリストの空要素除去後に真偽判定できます。

組み込みの value() サブテンプレートでの使用例（sql/sqlTemplate.stg）：

-- 値が空なら null、数値型ならそのまま、文字列型なら escapeSql で引用符付き出力
value(row, column) ::= <<$if(!DropEmpty.(row.(column.columnName)))$null$elseif(column.dataType.number)$$row.(column.columnName)$$else$$row.(column.columnName); format="escapeSql"$$endif$>>

-- 入力: value=""  → null
-- 入力: value=42  (数値) → 42
-- 入力: value="O'Brien" (文字列) → 'O''Brien'

主キーの有無で出力を切り替える例（settings/settingTemplate.stg）：

-- strip() でリストの空要素を除いて空でなければ主キーあり
setting(table) ::= <<
$if(strip(table.primaryKeys))$$usePrimaryKeys(table)$$else$$useColumns(table)$$endif$
>>

commit; を条件付きで出力する例（sql/sqlTemplate.stg）：

commit() ::= <<$if(inputParam.commit)$commit;$endif$>>

-- inputParam.commit=true  → commit;
-- inputParam.commit=false → （空文字）

サブテンプレート定義と呼び出し

.stg ファイルにサブテンプレートを定義し、本体 .txt や他のサブテンプレートから呼び出せます。引数を渡すことも可能です。

-- .stg（グループファイル）
insert() ::= <<
$rows:{row |insert into $tableName$ (...) values (...);
}$
>>

deleteAll() ::= <<
delete from $tableName$;
>>

commit() ::= <<$if(inputParam.commit)$commit;$endif$>>

-- 引数付きサブテンプレート
value(row, column) ::= <<...>>

setting(table) ::= <<
$if(strip(table.primaryKeys))$$usePrimaryKeys(table)$$else$$useColumns(table)$$endif$
>>
-- 本体 .txt でサブテンプレートを呼び出す（cleanInsertTemplate.txt の例）
$deleteAll()$

$insert()$$commit()$

-- 出力:
delete from USERS;

insert into USERS (id,name,age) values (1,'Alice',30);
insert into USERS (id,name,age) values (2,'O''Brien',25);

マップリテラルと間接参照

["キー": 値, default: 値] でマップを定義し、$マップ.(キー)$ でキーを変数として間接参照できます。空文字キーに空リスト [] を対応付けると、空値を falsy として扱う慣用パターンになります。

sql/sqlTemplate.stg での空値判定への応用：

-- 空文字 "" → [] (falsy)、それ以外 → キー自身 (truthy) にマップするテーブル
DropEmpty ::= ["": [], default: key]

-- $if(!DropEmpty.(row.(column.columnName)))$ で
-- row の当該カラム値が空文字のとき null を出力する
value(row, column) ::= <<$if(!DropEmpty.(row.(column.columnName)))$null$elseif(...)$...
>>

テンプレート内での書式指定

属性参照の後ろに ; format="書式名" を付けて、拡張書式をテンプレート本体で直接適用できます（サブテンプレート呼び出し構文 :書式名() とは別記法）。

sql/sqlTemplate.stg での使用例：

-- 文字列値を SQL エスケープしてシングルクォートで囲む
$row.(column.columnName); format="escapeSql"$

-- 入力: O'Brien
-- 出力: 'O''Brien'

ネストした繰り返し

外側の繰り返し変数をそのまま内側で参照できます。settings/settingTemplate.stg でのネスト例：

-- テーブルのリストを外側で回し、各テーブルのカラムを内側で回す
$dataSet.keys:{tableName |$setting(dataSet.(tableName))$}; separator="\n  , "$

-- setting() の中で primaryKeys・columns をさらに繰り返す
usePrimaryKeys(table) ::= <<
    {
      "name" : "$table.tableName$"
     ,"keys":[$table.primaryKeys:{column | "$column.columnName$"}; separator=" ,"$]
    }
>>

総合例：INSERT 文テンプレートの自作

上記パターンをすべて組み合わせて、数値・文字列・空値を正しく扱う INSERT テンプレートを自作する例です。

-- myInsert.stg（グループファイル）
DropEmpty ::= ["": [], default: key]

value(row, column) ::= <<$if(!DropEmpty.(row.(column.columnName)))$null$elseif(column.dataType.number)$$row.(column.columnName)$$else$$row.(column.columnName); format="escapeSql"$$endif$>>

insert() ::= <<
$rows:{row |insert into $tableName$ ($columns:{col | $col.columnName$}; separator=", "$) values ($columns:{col | $value(row, col)$}; separator=", "$);
}$
>>

commit() ::= <<$if(inputParam.commit)$
commit;$endif$
>>
-- myInsert.txt（本体）
$insert()$$commit()$
-- 入力: USERS テーブル（id=数値, name=文字列, age=数値）
-- 行1: id=1, name=Alice,   age=30
-- 行2: id=2, name=O'Brien, age=（空）

-- 出力（inputParam.commit=true）:
insert into USERS (id, name, age) values (1, 'Alice', 30);
insert into USERS (id, name, age) values (2, 'O''Brien', null);
commit;
