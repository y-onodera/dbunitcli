StringTemplate 4 書式リファレンス

DBUnit CLI のテンプレート処理では StringTemplate 4 (ST4) を使用します。テンプレート内の値は $値:書式名()$ の形式でフォーマットできます。ST4 標準の書式に加えて、DBUnit CLI 独自の拡張書式が利用可能です。

基本構文

書式は ST4 のレンダラー呼び出しとして、属性参照の直後に : で連結します。

$param.columnName:書式名()$

区切り文字が変更されている場合は、対応する文字で囲みます（例: <param.columnName:書式名()>）。詳細は Template の templateVarStart / templateVarStop を参照してください。

ST4 標準書式

ST4 の StringRenderer が提供する標準書式です。

書式名説明入力例出力例
upper大文字に変換helloHELLO
lower小文字に変換HELLOhello
cap先頭を大文字に変換hello worldHello world
url-encodeURL エンコードa ba+b
xml-encodeXML エンティティにエスケープ（& < 等）<tag>&lt;tag&gt;

DBUnit CLI 拡張書式（SqlEscapeStringRenderer）

DBUnit CLI は SqlEscapeStringRenderer により以下の拡張書式を追加しています。

命名規則変換

書式名説明入力例出力例
camelToSnakeキャメルケース → スネークケース（小文字）userIduser_id
camelToUpperSnakeキャメルケース → スネークケース（大文字）userIdUSER_ID
snakeToCamelスネークケース → キャメルケース（先頭小文字）user_iduserId
snakeToUpperCamelスネークケース → アッパーキャメルケース（先頭大文字）user_idUserId
camelToKebabキャメルケース → ケバブケース（小文字）userIduser-id
camelToUpperKebabキャメルケース → ケバブケース（大文字）userIdUSER-ID
kebabToCamelケバブケース → キャメルケース（先頭小文字）user-iduserId
kebabToUpperCamelケバブケース → アッパーキャメルケース（先頭大文字）user-idUserId

使用例（カラム名 userId を SQL の列定義に使う）:

ALTER TABLE $param.tableName$ ADD COLUMN $param.columnName:camelToSnake()$ VARCHAR(255);
-- 入力: tableName=userProfile, columnName=firstName
-- 出力: ALTER TABLE userProfile ADD COLUMN first_name VARCHAR(255);

SQL エスケープ

書式名説明
escapeSqlシングルクォートを '' にエスケープして SQL 文字列リテラルとして安全に出力する。4,000 バイト超の値は Oracle の TO_CLOB() に分割して連結。改行文字（\n / \r\n）は CHR(10) / CHR(13) || CHR(10) で連結

使用例:

INSERT INTO messages (body) VALUES ($param.body:escapeSql()$);
-- 入力: body = It's a test
-- 出力: INSERT INTO messages (body) VALUES ('It''s a test');

-- 入力: body = line1\nline2
-- 出力: INSERT INTO messages (body) VALUES ('line1' || CHR(10) || 'line2');

4,000 バイトを超える長文テキストの場合:

-- 出力例（4,001 バイトの文字列）:
-- TO_CLOB('...最初の4000バイト...') || TO_CLOB('...残りのバイト...')

式展開エスケープ

テンプレート内で別のテンプレートエンジンの式記法をそのまま出力したい場合に使います。

書式名説明入力例出力例
jexlExp値を JEXL 式 ${...} として出力user.name${user.name}
ST4Exp値を ST4 式 $...$ として出力（ST4 テンプレートを生成するテンプレートで使用）param.id$param.id$

使用例（ST4 テンプレートを生成する ST4 テンプレート）:

-- 入力: expr = param.tableName
-- テンプレート:
SELECT * FROM $param.expr:ST4Exp()$;

-- 出力（ST4 式が文字列としてそのまま埋め込まれる）:
SELECT * FROM $param.tableName$;

複合例

命名規則変換と SQL エスケープを組み合わせた例です。

テンプレート (insert.sql):
  INSERT INTO $param.tableName:camelToSnake()$
    ($param.col1:camelToSnake()$, $param.col2:camelToSnake()$)
  VALUES
    ($param.val1:escapeSql()$, $param.val2:escapeSql()$);

入力行:
  tableName = userProfile
  col1      = firstName
  col2      = lastName
  val1      = O'Brien
  val2      = Alice

レンダリング結果:
  INSERT INTO user_profile
    (first_name, last_name)
  VALUES
    ('O''Brien', 'Alice');

組み込みテンプレート

DBUnit CLI はよく使う生成パターンを組み込みテンプレートとして提供しています。Generate コマンドの template / templateGroup にこれらを指定することで、データセットから SQL や設定ファイルを即座に生成できます。

各テンプレートには .txt（本体）と .stg（共通マクロ定義グループ）が対になっています。template に .txt を、templateGroup に .stg を指定します。

INSERT 文生成（insertTemplate）

テーブルの各行を INSERT INTO 文に変換します。inputParam.commit が true のとき末尾に commit; を追加します。

設定項目値
templatesql/insertTemplate.txt
templateGroupsql/sqlTemplate.stg

入力データ（USERS テーブル、主キー: id）:

id | name    | age
---+---------+----
 1 | Alice   |  30
 2 | O'Brien |  25

生成結果:

insert into USERS (id ,name ,age ) values (1 ,'Alice' ,30 );
insert into USERS (id ,name ,age ) values (2 ,'O''Brien' ,25 );

文字列値はシングルクォートで囲まれ、' は自動的に '' にエスケープされます。空値は null に変換されます。

全削除 + INSERT 文生成（cleanInsertTemplate）

テーブルを全件削除してから INSERT します。テストデータ初期化に便利です。

設定項目値
templatesql/cleanInsertTemplate.txt
templateGroupsql/sqlTemplate.stg

生成結果（USERS テーブル）:

delete from USERS;

insert into USERS (id ,name ,age ) values (1 ,'Alice' ,30 );
insert into USERS (id ,name ,age ) values (2 ,'O''Brien' ,25 );

DELETE + INSERT 文生成（deleteInsertTemplate）

主キーで対象行を DELETE してから INSERT します。既存データを保持しつつ特定行を差し替えます。

設定項目値
templatesql/deleteInsertTemplate.txt
templateGroupsql/sqlTemplate.stg

生成結果（主キー: id）:

delete from USERS where id = 1 ;
delete from USERS where id = 2 ;

insert into USERS (id ,name ,age ) values (1 ,'Alice' ,30 );
insert into USERS (id ,name ,age ) values (2 ,'O''Brien' ,25 );

DELETE 文生成（deleteTemplate）

主キー条件で対象行を DELETE します。

設定項目値
templatesql/deleteTemplate.txt
templateGroupsql/sqlTemplate.stg

生成結果:

delete from USERS where id = 1 ;
delete from USERS where id = 2 ;

UPDATE 文生成（updateTemplate）

主キー以外のカラムを SET 句に、主キーを WHERE 句に展開した UPDATE 文を生成します。

設定項目値
templatesql/updateTemplate.txt
templateGroupsql/sqlTemplate.stg

生成結果（主キー: id）:

update USERS set name = 'Alice' ,age = 30 where id = 1 ;
update USERS set name = 'O''Brien' ,age = 25 where id = 2 ;

CREATE TABLE 生成（ddlTemplate）

テーブルのカラム定義行（カラム名・SQL 型・精度・スケール・NULL 制約・コメント）から CREATE TABLE 文を生成します。主キー情報があれば PRIMARY KEY 句も出力されます。

設定項目値
templatesql/ddlTemplate.txt
templateGroupsql/ddlTemplate.stg

入力データ（カラム定義行）:

columnName | sqlType  | precision | scale | nullable | comment
-----------+----------+-----------+-------+----------+----------
id         | NUMBER   | 10        | 0     | false    | 'ユーザーID'
name       | VARCHAR2 | 100       |       | false    | '氏名'
age        | NUMBER   | 3         | 0     | true     | '年齢'

生成結果（主キー: id）:

create table USERS (
id NUMBER(10,0) not null,
name VARCHAR2(100) not null,
age NUMBER(3,0)
comment on column USERS.id is 'ユーザーID'
comment on column USERS.name is '氏名'
comment on column USERS.age is '年齢'
,
primary key (id)
);

Dataset Setting 生成（settingTemplate）

テーブルのカラム情報から Dataset Settings の JSON を生成します。主キーがあればそれをキーに、なければ全カラムをキーとして設定します。

設定項目値
templatesettings/settingTemplate.txt
templateGroupsettings/settingTemplate.stg

入力: USERS テーブル（主キー: id、カラム: id, name, age）

生成結果（includeAllColumns=false の場合）:

{
  "settings": [
    {
      "name" : "USERS"
     ,"keys":["id"]
    }
  ]
}

生成結果（includeAllColumns=true の場合）:

{
  "settings": [
    {
      "name" : "USERS"
     ,"keys":["id"]
     ,"include":["id" ,"name" ,"age"]
     ,"exclude":[]
     ,"string":{
        "name":"name"
      }
     ,"number":{
        "id":"id"
       ,"age":"age"
      }
     ,"boolean":{}
     ,"sqlFunction":{}
    }
  ]
}

固定長カラム定義生成（fixedColumnDefTemplate）

固定長テキスト出力に必要なカラム幅・整列・パディング設定の JSON を生成します。

設定項目値
templatefixedcolumndef/fixedColumnDefTemplate.txt
templateGroupfixedcolumndef/fixedColumnDefTemplate.stg

入力データ（カラム定義行）:

name | length | align | pad
-----+--------+-------+----
id   | 4      | right | 0
name | 20     | left  |

生成結果:

{
  "columns": [
{"name": "id", "length": 4, "align": "right", "pad": "0"},{"name": "name", "length": 20, "align": "left", "pad": " "}
  ]
}

バッチパラメータ生成（generateParamTemplate）

Generate コマンドのバッチ実行用パラメータファイルを生成します。Parameterize コマンドと組み合わせ、複数の Generate 実行を自動化する場面で使います。

設定項目値
templateparam/generateParamTemplate.txt

テンプレート内容:

-srcType=$srcType$
-src=$src$
-encoding=$encoding$
-setting=$setting$
-generateType=$generateType$
-template=$template$
-templateGroup=$templateGroup$
-templateEncoding=$templateEncoding$
-outputEncoding=$outputEncoding$
-resultPath=$resultPath$

入力行（1 件）:

srcType=csv
src=data/users.csv
encoding=UTF-8
setting=settings/users.json
generateType=txt
template=sql/insertTemplate.txt
templateGroup=sql/sqlTemplate.stg
templateEncoding=UTF-8
outputEncoding=UTF-8
resultPath=output/$\{tableName\}.sql

生成結果（1 ファイル）:

-srcType=csv
-src=data/users.csv
-encoding=UTF-8
-setting=settings/users.json
-generateType=txt
-template=sql/insertTemplate.txt
-templateGroup=sql/sqlTemplate.stg
-templateEncoding=UTF-8
-outputEncoding=UTF-8
-resultPath=output/${tableName}.sql
