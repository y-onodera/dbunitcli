JDBC

データベースへ接続するための設定を指定します。以下の場面で必要になります。

  • Source Type = table（テーブル名リストから読み込む）
  • Source Type = sql（SQL クエリ結果を読み込む）
  • Result Type = table（Compare / Convert / Generate の結果をテーブルに出力）
  • Run の Script Type = sql（SQL スクリプトを DB で実行）

設定項目

フォームの jdbc option 展開ボタンから設定します。各項目は個別に入力しても、jdbcProperties に保存済みの .properties ファイルを選んでも構いません。両方が指定された場合、jdbcUrl / jdbcUser / jdbcPass が jdbcProperties の同名キーを上書きします。

項目説明
jdbcProperties接続情報を記述した .properties ファイルのパス。中身は url= / user= / pass= の 3 行。ワークスペースの jdbc/ ディレクトリ配下を既定の参照先とします。
jdbcUrlJDBC 接続 URL（例: jdbc:postgresql://localhost:5432/mydb）。jdbcProperties の url を上書きします。
jdbcUserログインユーザ名。jdbcProperties の user を上書きします。
jdbcPassログインパスワード。jdbcProperties の pass を上書きします。
useJdbcMetaDataDBUnit のメタデータ取得経路を切り替えるフラグ（table / sql の Data Load オプション）。デフォルトは OFF。詳細は Dataset Load Form の table の節を参照。

.properties ファイルの書式

Java 標準の Properties 形式。以下 3 つのキーを定義します。

url=jdbc:postgresql://localhost:5432/mydb
user=app_user
pass=app_password

JDBC URL Builder

jdbcUrl テキストボックス横の編集ボタンから開くダイアログです。代表的な RDB の URL テンプレートに従って URL を組み立てます。Host / Port / Service Name (Database Name) を入力すると、下部の Preview に完成 URL が表示され、Apply で jdbcUrl に反映されます。

Database既定ポートURL テンプレート
oracle1521jdbc:oracle:thin:@//{host}:{port}/{serviceName}
postgres5432jdbc:postgresql://{host}:{port}/{serviceName}
h29092jdbc:h2:tcp://{host}:{port}/{serviceName}

Database ラベル（Service Name / Database Name）は RDB 種別に応じて切り替わります（Oracle は Service Name、PostgreSQL / H2 は Database Name）。既存 URL を開いた場合、Builder はその URL を解析してフォーム項目を初期値に復元します。

Connection Test

フォーム下部の Connection Test ボタンを押すと sidecar 経由で実際の接続を試し、結果（成功 / 失敗メッセージ）を隣に表示します。以下のいずれかが揃っていると押下可能です。

  • jdbcUrl + jdbcUser + jdbcPass がすべて入力されている
  • jdbcProperties が選択されている

フォーム値を変更すると接続 OK 状態は解除され、再度テストが必要になります。

Save as Properties

現在入力中の URL / User / Pass を新規 .properties ファイルとしてワークスペースの jdbc/ に保存します。保存後は jdbcProperties で再利用できます。ダイアログには保存される内容のプレビュー（Pass はマスク）とファイル名入力欄が表示されます。
