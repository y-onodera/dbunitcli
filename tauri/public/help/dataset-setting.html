Dataset Setting

個別テーブルに対するメタデータ設定を編集するダイアログです。対象指定、分割・リネーム、派生テーブルの分離、追加カラム、フィルタ／並び順を構成します。ファイル全体の構造（settings / commonSettings / import）については Dataset Settings を参照してください。

Target

処理対象のテーブルをどのように特定するかを選択します。

種別説明
nameテーブル名を1件以上列挙して指定します。filePath で設定ファイルからの相対パスを指定することも可能です。
patternテーブル名のマッチで対象を特定します。pattern オブジェクトには部分一致用の string または正規表現用の regex を指定し、exclude 配列で除外パターンを追加できます。
outerJoin / innerJoin / fullJoin2テーブルを結合して1つの論理テーブルとして扱います。left と right に結合対象のテーブル名、条件は on（JeXL 式）か column（共通カラム名）で指定します。

変化の例

name（対象テーブルの絞り込み）

入力に users、orders、logs がある状態で name: ["users", "orders"] を指定すると、logs は処理対象から外れます。

pattern（正規表現マッチ）

pattern: { string: "t_.*", exclude: ["t_tmp_.*"] } を指定すると、t_users や t_orders は対象になり、t_tmp_work は除外されます。

innerJoin（内部結合）

Before (orders):
order_id | item_id | qty
---------+---------+----
1        | 101     | 2
2        | 102     | 3
3        | 999     | 1

Before (items):
id  | name
----+------
101 | Pen
102 | Book

After innerJoin (left=orders, right=items, column=["item_id","id"]):
orders_order_id | orders_item_id | orders_qty | items_id | items_name
----------------+----------------+------------+----------+-----------
1               | 101            | 2          | 101      | Pen
2               | 102            | 3          | 102      | Book

outerJoin は左側（orders）の全行を残し、一致しない右側カラムは NULL になります。fullJoin は両側の未一致行も残します。

Split / Rename

split を有効にすると、1つの入力から複数テーブルを派生させることができます。無効時はリネーム設定として機能します。

フィールド説明
prefix出力テーブル名に付与する接頭辞
tableName出力テーブル名のベース名
suffix出力テーブル名に付与する接尾辞
limit1 出力テーブルあたりに含める breakKey のグループ数（breakKey 未指定時は行数）の上限。上限に達すると次の行から新しい出力テーブルへ切り替わる。0（未指定）のとき split は行われない。
breakKeysplit 時にテーブルを切り替えるキーとなるカラム。値が変化した回数が limit に達すると出力テーブルが切り替わる。未指定の場合は limit 行ごとに切り替わる。

変化の例

prefix / tableName / suffix（リネーム）

入力 users に対し prefix: "tmp_"、tableName: "accounts"、suffix: "_v1" を指定すると、出力テーブル名は tmp_accounts_v1 になります。データ内容はそのまま維持されます。

split（breakKey でグループ単位に分割）

出力テーブル名は prefix / suffix に %d を含めることで、分割順の連番（0, 1, 2, ...）が埋め込まれます。breakKey の値自体はテーブル名には使われません。

Before (orders, breakKey 昇順にソート済み):
id | type | amount
---+------+-------
1  | A    | 100
3  | A    | 150
2  | B    | 200
4  | C    | 80

After (split: { tableName: "orders", suffix: "_%d", breakKey: ["type"], limit: 1 }):
Table: orders_0
id | type | amount
---+------+-------
1  | A    | 100
3  | A    | 150

Table: orders_1
id | type | amount
---+------+-------
2  | B    | 200

Table: orders_2
id | type | amount
---+------+-------
4  | C    | 80

split.limit（1 テーブルに含めるグループ数の上限）

limit は「1 出力テーブルあたりに含める breakKey のグループ数の上限」です。上限に達すると、それ以降の行は新しい出力テーブルに振り分けられます（行は破棄されません）。上記と同じ入力に対して limit: 2 を指定すると、2 グループ分（A と B）が 1 テーブル目に、残りのグループ（C）が 2 テーブル目に格納されます。

After (split: { tableName: "orders", suffix: "_%d", breakKey: ["type"], limit: 2 }):
Table: orders_0
id | type | amount
---+------+-------
1  | A    | 100
3  | A    | 150
2  | B    | 200

Table: orders_1
id | type | amount
---+------+-------
4  | C    | 80

breakKey を指定しない場合、limit は「1 出力テーブルあたりの行数」として扱われ、limit 行ごとに新しい出力テーブルへ切り替わります。

Separate

separate は配列で、各要素が 1 つの派生テーブル定義となり、1 つの入力から複数の派生テーブルを並列に生成します。split が breakKey / limit で連番テーブルを自動生成するのに対し、separate は派生テーブルごとに列・行・並び順などを明示的に自由定義できる点が異なります。

各派生テーブルでは、通常のテーブル設定と同じフィールドを指定できます。include / exclude による列方向の切り出しと、filter / distinct による行方向の切り出しの両方に利用でき、1 要素の中で同時に指定することも可能です。

フィールド説明
name派生テーブルの出力テーブル名
keys派生テーブルの主キーとなるカラム
include派生テーブルに残すカラム（列の切り出し）。指定すると include 以外は除外される。
exclude派生テーブルから除外するカラム
filter派生テーブルに残す行の条件（JeXL 式）。複数指定した場合は AND 扱い。
order派生テーブル内の並び順に使用するカラム
distinct派生テーブルの重複行を排除するフラグ
string / number / boolean / function追加カラム定義。派生テーブルごとに個別に指定できる（詳細は Additional Columns を参照）。

変化の例

列方向の切り出し（連絡先カラムを別テーブルへ）

入力 users から連絡先関連のカラムだけを user_profiles に切り出します。元テーブル users 自体は separate の対象外になるため、連絡先カラムを残したい場合は別途 settings を追加してください。

Before (users):
user_id | name  | address       | tel          | fax
--------+-------+---------------+--------------+--------------
1       | Alice | Tokyo         | 03-0000-0001 | 03-0000-0002
2       | Bob   | Osaka         | 06-0000-0001 | 06-0000-0002

After (separate: [{ name: "user_profiles", keys: ["user_id"], include: ["user_id", "address", "tel", "fax"] }]):
Table: user_profiles
user_id | address | tel          | fax
--------+---------+--------------+--------------
1       | Tokyo   | 03-0000-0001 | 03-0000-0002
2       | Osaka   | 06-0000-0001 | 06-0000-0002

行方向の切り出し（ステータス別に派生テーブルを作成）

separate 配列に複数要素を定義することで、同じ入力から異なる条件の派生テーブルを並列に生成できます。

Before (orders):
id | status   | amount
---+----------+-------
1  | APPROVED | 1500
2  | REJECTED | 500
3  | APPROVED | 2000
4  | REJECTED | 300

After (separate: [
  { name: "orders_approved", keys: ["id"], filter: ["status == 'APPROVED'"] },
  { name: "orders_rejected", keys: ["id"], filter: ["status == 'REJECTED'"] }
]):
Table: orders_approved
id | status   | amount
---+----------+-------
1  | APPROVED | 1500
3  | APPROVED | 2000

Table: orders_rejected
id | status   | amount
---+----------+-------
2  | REJECTED | 500
4  | REJECTED | 300

列方向 + 行方向の組み合わせ

1 つの派生テーブル定義の中で include と filter を同時に指定すると、「条件に合致する行」の「特定カラムだけ」を切り出せます。

Before (orders):
id | status   | amount | memo
---+----------+--------+------
1  | APPROVED | 1500   | x
2  | REJECTED | 500    | y
3  | APPROVED | 2000   | z

After (separate: [{ name: "orders_approved_summary", keys: ["id"], include: ["id", "amount"], filter: ["status == 'APPROVED'"] }]):
Table: orders_approved_summary
id | amount
---+-------
1  | 1500
3  | 2000

Additional Columns

元データに存在しないカラムを追加します。値には JeXL 式を記述できます。

型説明
string文字列型カラム。既存カラムの連結や substring による抽出に利用します。
number数値型カラム。算術演算で集計値や派生値を定義します。
boolean真偽値型カラム。条件式で判定結果を持たせます。
function関数表現によるカラム。複雑なロジックを JeXL 関数として定義します。

変化の例

string（文字列連結）

Before (users):
id | firstName | lastName
---+-----------+---------
1  | John      | Smith
2  | Anna      | Lee

After (string: { fullName: "firstName + ' ' + lastName" }):
id | firstName | lastName | fullName
---+-----------+----------+-----------
1  | John      | Smith    | John Smith
2  | Anna      | Lee      | Anna Lee

number（算術による派生カラム）

Before (items):
id | price | quantity
---+-------+---------
1  | 100   | 3
2  | 250   | 2

After (number: { subtotal: "price * quantity" }):
id | price | quantity | subtotal
---+-------+----------+---------
1  | 100   | 3        | 300
2  | 250   | 2        | 500

boolean（条件式による判定）

Before (users):
id | age
---+----
1  | 25
2  | 17

After (boolean: { adult: "age >= 20" }):
id | age | adult
---+-----+------
1  | 25  | true
2  | 17  | false

function（条件分岐による区分値）

Before (orders):
id | status
---+---------
1  | APPROVED
2  | REJECTED
3  | PENDING

After (function: { mark: "status == 'APPROVED' ? 'OK' : (status == 'REJECTED' ? 'NG' : 'HOLD')" }):
id | status   | mark
---+----------+-----
1  | APPROVED | OK
2  | REJECTED | NG
3  | PENDING  | HOLD

Filter / Order

フィールド説明
exclude処理から除外するカラム
include処理対象にするカラム（指定すると include 以外は除外）
keys主キーのカラム名
filterデータ選択条件（JeXL 式）。複数指定した場合は AND 扱い。
order並び順に使用するカラム

変化の例

exclude（カラム除外）

Before (users):
id | name  | created_at          | updated_at
---+-------+---------------------+---------------------
1  | Alice | 2024-01-01 10:00:00 | 2024-01-02 09:00:00

After (exclude: ["created_at", "updated_at"]):
id | name
---+------
1  | Alice

include（対象カラムの限定）

Before (users):
id | name  | email       | age
---+-------+-------------+----
1  | Alice | a@example   | 25

After (include: ["id", "name"]):
id | name
---+------
1  | Alice

keys（主キー指定）

カラム構成や行数は変わりません。比較コマンドでの突合キーや、distinct・ソート挙動の基準として使われます。

filter（行のフィルタ）

Before (orders):
id | amount | status
---+--------+---------
1  | 1500   | APPROVED
2  | 500    | APPROVED
3  | 2000   | REJECTED

After (filter: ["amount > 1000 && status == 'APPROVED'"]):
id | amount | status
---+--------+---------
1  | 1500   | APPROVED

order（並び替え）

Before (users):
id | name
---+------
3  | Carol
1  | Alice
2  | Bob

After (order: ["id"]):
id | name
---+------
1  | Alice
2  | Bob
3  | Carol

JeXL 式の書き方

filter、追加カラム（string / number / boolean / function）、結合条件 on では Apache Commons JEXL3 の式を利用できます。

演算子

カテゴリ演算子
比較==, !=, >, <, >=, <=
論理&& (AND), || (OR), ! (NOT)
算術+, -, *, /, %
文字列連結+
三項condition ? valueIfTrue : valueIfFalse
NULL 比較== null, != null

文字列メソッド

  • contains(str) — 指定文字列を含むか
  • startsWith(prefix) — 指定文字列で始まるか
  • endsWith(suffix) — 指定文字列で終わるか
  • matches(regex) — 正規表現にマッチするか
  • substring(start, end) — 部分文字列の取得

文字列リテラルはシングルクォートで囲みます（例: 'APPROVED'）。

記述例

filter（金額とステータスの複合条件）

amount > 1000 && status == 'APPROVED'

filter（NULL 判定と前方一致）

type != null && type.startsWith('SALE')

boolean（カラム名の接尾辞で判定）

col.endsWith('_amount')

string（フルネーム連結）

firstName + ' ' + lastName

string（日付の年月日部分を抽出）

created_at.substring(0, 10)

number（小計の算出）

price * quantity

boolean（三項演算子による区分判定）

age >= 20 ? true : false

function（条件に応じた区分値の生成）

status == 'APPROVED' ? 'OK' : (status == 'REJECTED' ? 'NG' : 'PENDING')

join.on（結合条件）

orders_item_id == items_id

結合時のカラム参照は <テーブル名>_<カラム名> の形式を使用します。

注意事項

  1. 文字列比較は大文字・小文字を区別します
  2. 日付比較は文字列として行われます（必要ならフォーマットを揃えてから比較）
  3. NULL との比較は必ず == null / != null を使用
  4. 複雑な条件は filter を複数行に分割して AND 合成することを推奨
