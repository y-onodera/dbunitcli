Dataset Setting

個別テーブルに対するメタデータ設定を編集するダイアログです。対象指定、分割・リネーム、追加カラム、フィルタ／並び順を構成します。ファイル全体の構造（settings / commonSettings / import）については Dataset Settings を参照してください。

Target

処理対象のテーブルをどのように特定するかを選択します。

種別説明
nameテーブル名を1件以上列挙して指定します。filePath で設定ファイルからの相対パスを指定することも可能です。
patternテーブル名の正規表現で対象を特定します。patternExclue で除外パターンを追加できます。
outerJoin / innerJoin / fullJoin2テーブルを結合して1つの論理テーブルとして扱います。left と right に結合対象のテーブル名、条件は on（JeXL 式）か column（共通カラム名）で指定します。

Split / Rename

split を有効にすると、1つの入力から複数テーブルを派生させることができます。無効時はリネーム設定として機能します。

フィールド説明
prefix出力テーブル名に付与する接頭辞
tableName出力テーブル名のベース名
suffix出力テーブル名に付与する接尾辞
limitsplit 時の最大分割数
breakKeysplit 時にテーブルを分割するキーとなるカラム

Additional Columns

元データに存在しないカラムを追加します。値には JeXL 式を記述できます。

型説明
string文字列型カラム。既存カラムの連結や substring による抽出に利用します。
number数値型カラム。算術演算で集計値や派生値を定義します。
boolean真偽値型カラム。条件式で判定結果を持たせます。
function関数表現によるカラム。複雑なロジックを JeXL 関数として定義します。

Filter / Order

フィールド説明
exclude処理から除外するカラム
include処理対象にするカラム（指定すると include 以外は除外）
keys主キーのカラム名
filterデータ選択条件（JeXL 式）。複数指定した場合は AND 扱い。
order並び順に使用するカラム

JeXL 式の書き方

filter、追加カラム（string / number / boolean / function）、結合条件 on では Apache Commons JEXL3 の式を利用できます。

演算子

カテゴリ演算子
比較==, !=, >, <, >=, <=
論理&& (AND), || (OR), ! (NOT)
算術+, -, *, /, %
文字列連結+
三項condition ? valueIfTrue : valueIfFalse
NULL 比較== null, != null

文字列メソッド

  • contains(str) — 指定文字列を含むか
  • startsWith(prefix) — 指定文字列で始まるか
  • endsWith(suffix) — 指定文字列で終わるか
  • matches(regex) — 正規表現にマッチするか
  • substring(start, end) — 部分文字列の取得

文字列リテラルはシングルクォートで囲みます（例: 'APPROVED'）。

記述例

filter（金額とステータスの複合条件）

amount > 1000 && status == 'APPROVED'

filter（NULL 判定と前方一致）

type != null && type.startsWith('SALE')

boolean（カラム名の接尾辞で判定）

col.endsWith('_amount')

string（フルネーム連結）

firstName + ' ' + lastName

string（日付の年月日部分を抽出）

created_at.substring(0, 10)

number（小計の算出）

price * quantity

boolean（三項演算子による区分判定）

age >= 20 ? true : false

function（条件に応じた区分値の生成）

status == 'APPROVED' ? 'OK' : (status == 'REJECTED' ? 'NG' : 'PENDING')

join.on（結合条件）

orders_item_id == items_id

結合時のカラム参照は <テーブル名>_<カラム名> の形式を使用します。

注意事項

  1. 文字列比較は大文字・小文字を区別します
  2. 日付比較は文字列として行われます（必要ならフォーマットを揃えてから比較）
  3. NULL との比較は必ず == null / != null を使用
  4. 複雑な条件は filter を複数行に分割して AND 合成することを推奨
