Image Option

Compare で targetType に image または pdf を選択した場合に表示される、画像差分オプションです。比較エンジンには image-comparison ライブラリを使用します。

入力には Source Type = file を用いてディレクトリを走査し、拡張子フィルタ（例: .png、.pdf）で対象ファイルを絞り込みます。Compare は oldData と newData のファイルを 1 対 1 で突き合わせ、差分画像と判定結果を出力します。

設定項目

フォームの image セクションに表示される項目の一覧です。すべて任意で、未指定時は以下の既定値が使われます。

比較しきい値

項目型説明既定
threshold整数2 ピクセルが異なるとみなす RGB 距離のしきい値。大きくするほど差分に寛容5
pixelToleranceLevel小数 (0.0–1.0)ピクセル単位の許容レベル。例: 0.1 は 10% の差まで同一とみなす0.01
allowingPercentOfDifferentPixels小数 (0.0–1.0)画像全体で MATCH 判定を維持できる差分ピクセル比率。これを超えると MISMATCH になる0.01

差分矩形の描画

差分があった領域は、出力される差分画像上で矩形として強調描画されます。

項目型説明既定
rectangleLineWidth整数 (px)矩形枠線の幅1
minimalRectangleSize整数 (px)描画する矩形の最小サイズ。小さな差分を無視したいときに利用1
maximalRectangleCount整数描画する矩形の最大数。-1 は無制限-1
fillDifferenceRectangles真偽値差分矩形を塗りつぶす（枠だけでなく中を塗る）false
percentOpacityDifferenceRectangles小数 (0–100)差分矩形の塗りの不透明度（%）20.0
differenceRectangleColor色差分矩形の色（後述「色の指定」参照）red

除外領域

比較から特定の矩形領域を除外できます。タイムスタンプや動的に変わる UI パーツなど、差分を許容したい領域を指定するのに使います。

項目型説明既定
excludedAreas文字列除外する矩形のリスト。書式は [x1,y1,x2,y2] を 0 個以上連結（後述）空（除外なし）
drawExcludedRectangles真偽値除外矩形の枠線を差分画像に描画するtrue
fillExcludedRectangles真偽値除外矩形を塗りつぶすfalse
percentOpacityExcludedRectangles小数 (0–100)除外矩形の塗りの不透明度（%）20.0
excludedRectangleColor色除外矩形の色green

excludedAreas の書式

1 つの矩形は [x1,y1,x2,y2]（画像の左上原点、x1,y1 は左上隅、x2,y2 は右下隅、単位はピクセル）で表します。複数指定する場合は区切り文字なしで連結します。

例: 2 つの矩形を除外する
  [10,20,100,150][200,50,300,200]

意味:
  (10,20)-(100,150)  の矩形と (200,50)-(300,200) の矩形を除外

色の指定

色は Java の Color.getColor() が解釈する形式で指定します。代表的な書き方は 16 進リテラル（例: 0xFF0000 で赤、0x00FF00 で緑、0x0000FF で青）です。未指定または解釈に失敗した場合は、差分矩形は red、除外矩形は green にフォールバックします。

出力イメージ

Compare の差分レポートには、以下 3 種類の矩形が重ねて描画された差分画像が含まれます。

  • 差分矩形 — 実際にピクセルが異なる領域（既定: 赤枠）
  • 除外矩形 — excludedAreas で指定した無視領域（既定: 緑枠）
  • 各矩形は fill*Rectangles と percentOpacity* の組み合わせで半透明に塗りつぶし可能
