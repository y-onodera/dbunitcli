Dataset Load Form

各コマンドで「データセットの入力」を指定するための共通フォームです。Compare の oldData / newData / expectData、Convert / Generate / Run の srcData、Parameterize の paramData は、いずれもこのフォームによって構成されます。

1 つの入力は、以下の 4 つの領域から構成されます。

  • Source Type 選択 — 入力の形式（CSV / Table / SQL / Regex / Fixed / Xls / Xlsx）
  • Source — 入力元ファイル・SQL、文字コード、ディレクトリ走査条件
  • Data Load — ヘッダ・開始行・メタデータなど、読み込み時の挙動
  • Setting — Dataset Settings ファイルによるテーブル単位のメタデータ指定

Source Type 選択

入力データの形式を選択します。選択した形式に応じて、固有の設定項目がフォーム下部に表示されます。以下の表は早見表です。各形式の詳細（src の意味・固有オプション・データイメージ例）は、形式名のリンクから対応する節を参照してください。

形式説明
none入力なし（空のデータセット）。入力データが不要なコマンド（テンプレート専用 Generate など）で使用
csvCSV / TSV など区切り文字付きテキスト。区切り文字・囲み文字を指定可能
fixed固定長テキスト。カラム幅（バイト数）と headerName を指定して列に分割
reg正規表現でヘッダ行・データ行をパースして列に分割
xls / xlsxExcel ファイル。シート単位でテーブル化
tableテーブル名を列挙したテキストファイルに従い、JDBC 接続先のテーブルを読み込む
sqlSQL クエリの実行結果。JDBC 接続設定と SQL ファイルを指定
csvqH2 上で SQL を実行した結果を取得。SQL 内の CSVREAD() 関数で CSV ファイルを参照し、複数 CSV の JOIN などに使用
fileディレクトリ配下のファイル一覧を、ファイルメタ情報（パス・サイズ・更新日時など）のテーブルとして読み込む
dirディレクトリ配下のサブディレクトリ一覧を、メタ情報のテーブルとして読み込む

各節は「src の意味 / テーブル名の決まり方 / 固有オプション / データイメージ例」の順で構成されています。共通オプション（encoding・headerName・startRow・addFileInfo・ディレクトリ走査系）は後続の Source / Data Load 節を参照してください。

csv

src の意味: CSV / TSV ファイル、または複数 CSV を含むディレクトリ。

テーブル名: 拡張子を除いたファイル名。

固有オプション:

  • delimiter — 区切り文字を 1 文字で指定します。デフォルトは ,（カンマ）。タブ・改行など制御文字はエスケープ記法で指定可能で、\b / \t / \n / \r / \f がサポートされます。
  • ignoreQuoted — 引用符（"）の扱いを切り替えます。デフォルトは OFF。
    • OFF（既定）: 引用符をクォート文字として扱う。フィールドを囲む " は除去され、引用符内の区切り文字はフィールドの一部となる
    • ON: 引用符を特別扱いせず、ただの文字として保持する
    値に区切り文字や改行を含む可能性がある通常の CSV では OFF のままにしてください。ON は、引用符をリテラルとして保持したい特殊ファイル向けです。

データイメージ例: 1 行目をヘッダ、2 行目以降をデータとして 1 つのテーブルを生成。

入力 (users.csv):
id,name,age
1,Alice,30
2,Bob,25

テーブル名: users  (拡張子を除いたファイル名)
カラム:     [id, name, age]
データ行:
  [1, Alice, 30]
  [2, Bob,   25]

delimiter の例（TSV を読む）:

入力 (users.tsv):
id	name	age
1	Alice	30
2	Bob	25

指定:
  srcType   = csv
  delimiter = \t     (タブ区切り)

カラム:     [id, name, age]
データ行:
  [1, Alice, 30]
  [2, Bob,   25]

ignoreQuoted の例:

入力 (multi.csv):
key,column1,column2
1,"a,b",3
2,"あいうえお",5

ignoreQuoted = OFF (既定):
  カラム:     [key, column1, column2]
  データ行:
    [1, a,b,           3]        (外側の " は除去、内側のカンマは値の一部)
    [2, あいうえお,     5]

ignoreQuoted = ON:
  カラム:     [key, column1, column2, (余分な列)]
  データ行:
    [1, "a, b", 3, ...]          (" がそのまま残り、"a,b" は 2 列に分割される)
    [2, "あいうえお", 5]

fixed

src の意味: 固定長テキストファイル、または複数ファイルを含むディレクトリ。

テーブル名: 拡張子を除いたファイル名。

固有オプション:

  • fixedLength — 各カラムのバイト幅をカンマ区切りで指定（例: 4,10）。headerName による列名指定との併用が必須です。

データイメージ例: 指定したバイト幅でカラム分割。

入力 (members.txt, fixedLength=4,10):
0001Alice
0002Bob

指定:
  fixedLength = 4,10
  headerName  = id,name

カラム:     [id, name]
データ行:
  [0001, Alice     ]
  [0002, Bob       ]

reg

src の意味: テキストファイル、または複数ファイルを含むディレクトリ。

テーブル名: 拡張子を除いたファイル名。

固有オプション:

  • regHeaderSplit — ヘッダ行を列に分割する正規表現。
  • regDataSplit — データ行を列に分割する正規表現。ヘッダと区切りロジックを変えたい場合（例: 引用符内のカンマを区切らない等）に使用します。

データイメージ例:

入力 (dirty.txt):
No,Contents,Time
1,"最小値1～最大値999,999,999まで",2019-04-01 12:22:30
2,"文言A,文言B",2019-04-02 09:00:00

指定:
  regHeaderSplit = ,
  regDataSplit   = ,(?=(?:[^"]*"[^"]*")*[^"]*$)

カラム:     [No, Contents, Time]
データ行:
  [1, 最小値1～最大値999,999,999まで, 2019-04-01 12:22:30]
  [2, 文言A,文言B,                    2019-04-02 09:00:00]

xls / xlsx

src の意味: Excel ファイル（.xls / .xlsx）、または複数 Excel を含むディレクトリ。

テーブル名: 各シート名（ブック内の各シートが 1 テーブル）。

固有オプション:

  • xlsxSchema — シートごとの列定義や任意セルの抽出定義を記述したスキーマファイル。書式の詳細は Xlsx Schema を参照してください。

データイメージ例: 先頭行がヘッダ、以降がデータ（既定）。

入力 (book.xlsx):
  Sheet "Users":   id,name / 1,Alice / 2,Bob
  Sheet "Orders":  id,amount / 10,500 / 11,800

テーブル "Users":
  カラム:   [id, name]
  データ行: [1, Alice] / [2, Bob]
テーブル "Orders":
  カラム:   [id, amount]
  データ行: [10, 500] / [11, 800]

table

src の意味: 読み込み対象のテーブル名を 1 行 1 件で列挙したテキストファイル、または同種のファイルを含むディレクトリ。ファイル自体は DB テーブルの名前ではなく、テーブル名のリストを記述したテキストファイルです。

テーブル名: src ファイル内の各行に書かれたテーブル名がそのまま DBUnit 上のテーブル名になります。jdbcProperties（JDBC 接続設定）が必須です。

固有オプション:

  • useJdbcMetaData — DBUnit のメタデータ取得方法を切り替えるフラグ。デフォルトは OFF。
    • OFF（既定）: connection.createTable(tableName) を使用。対象テーブルのメタデータのみを JDBC から取得する軽量な経路
    • ON: connection.createDataSet().getTable(tableName) を使用。接続先スキーマ全体のデータセットを構築したうえで対象テーブルを取り出す。DBUnit 内部でのカラム型解決やプライマリキー情報の取得精度が上がる一方、スキーマが大きいと初期化コストが増える
    通常は OFF で十分です。DB の種類や方言によってデフォルト経路でカラム型が想定と異なる場合にのみ ON を検討してください。

注: headerName / startRow / addFileInfo は DB テーブルが対象のため無効です。src ファイル（テーブル名リスト）自体は空行・コメントを含まない前提で読み込まれます。

データイメージ例: テーブル名リストファイル 1 件から複数テーブルを生成。

入力 (tables.txt):
USERS
ORDERS

JDBC 接続先:
  USERS(id INT, name VARCHAR, age INT)
    (1, 'Alice', 30), (2, 'Bob', 25)
  ORDERS(id INT, amount INT)
    (10, 500), (11, 800)

指定:
  srcType = table
  src     = tables.txt

生成テーブル:
  テーブル名: USERS
    カラム:     [id, name, age]
    データ行:   [1, Alice, 30] / [2, Bob, 25]
  テーブル名: ORDERS
    カラム:     [id, amount]
    データ行:   [10, 500] / [11, 800]

sql

src の意味: SQL クエリを記述したファイル、または複数 SQL を含むディレクトリ。jdbcProperties（JDBC 接続設定）が必須です。

テーブル名: SQL ファイル名（拡張子除く）。

固有オプション:

  • useJdbcMetaData — 意味は table と同じ。DBUnit のメタデータ取得経路を切り替えます。詳細は table の節を参照してください。

注: headerName / startRow / addFileInfo は SQL 実行結果には適用されず無効です。

データイメージ例:

入力 (active_users.sql):
  SELECT id, name FROM USERS WHERE active = 1

JDBC 実行結果:
  id=1 name=Alice
  id=3 name=Carol

テーブル名: active_users
カラム:     [id, name]
データ行:
  [1, Alice]
  [3, Carol]

csvq

src の意味: H2 データベース（ファイルモード）上で実行する SQL ファイル。どの CSV を読むかも SQL ファイル内で CSVREAD() 関数により指定します（src が指す先は SQL ファイルであり、CSV を置いたディレクトリではありません）。

テーブル名: SQL ファイル名（拡張子除く）。

固有オプション: なし。

データイメージ例: 複数 CSV を SQL で JOIN した結果を 1 テーブルとして取得。

入力 CSV (csvquery/csv/header.csv):
KEYCOLUMN,CD1,NAME,VALUE
1,A1,太郎,10000
2,B1,次郎,20000
3,C1,花子,30000

入力 CSV (csvquery/csv/detail.csv):
KEYCOLUMN,DETAILNO,VALUE1,VAL
1,1,100,200
1,2,200,400
2,1,0,0
3,2,1002,2001

SQL (joinQuery.sql):
  SELECT
      table1.*
  FROM
      CSVREAD('csvquery/csv/header.csv', null, 'charset=UTF-8') table1
      INNER JOIN
      CSVREAD('csvquery/csv/detail.csv', null, 'charset=UTF-8') table2
      ON table1.KEYCOLUMN = table2.KEYCOLUMN
  WHERE
      table2.DETAILNO = 2
  ORDER BY
      table1.KEYCOLUMN DESC;

指定:
  srcType = csvq
  src     = joinQuery.sql

テーブル名: joinQuery
カラム:     [KEYCOLUMN, CD1, NAME, VALUE]
データ行:
  [3, C1, 花子, 30000]
  [1, A1, 太郎, 10000]

CSVREAD(fileName, columns, csvOptions) の第 2 引数を null にすると先頭行がヘッダとして扱われます。第 3 引数では charset・fieldSeparator・fieldDelimiter などの H2 CSV オプションを指定できます（例: TSV を読む場合は 'charset=UTF-8 fieldSeparator=\t'）。

file

src の意味: 走査対象のディレクトリ。

テーブル名: 対象ファイルの拡張子（extension でフィルタした場合はその拡張子名）。

固有オプション: なし。ディレクトリ走査系（extension・recursive・regInclude・regExclude）で対象を絞り込みます。

データイメージ例: ディレクトリ配下のファイル一覧をメタ情報テーブルとして生成。

入力 (src = project/):
  project/a.txt         (2KB, 2026-04-01 10:00:00)
  project/sub/b.txt     (5KB, 2026-04-02 11:00:00)
  project/sub/c.log     (1KB, 2026-04-03 09:30:00)

指定:
  srcType   = file
  recursive = true
  extension = .txt

テーブル名: txt
カラム:     [PATH, NAME, DIR, RELATIVE_PATH, SIZE_KB, LAST_MODIFIED]
データ行:
  [/abs/project/a.txt,       a.txt, /abs/project,     a.txt,       2, 2026/04/01 10:00:00]
  [/abs/project/sub/b.txt,   b.txt, /abs/project/sub, sub/b.txt,   5, 2026/04/02 11:00:00]

dir

src の意味: 走査対象のディレクトリ。

テーブル名: ルートディレクトリ名。

固有オプション: なし。ディレクトリ走査系（recursive・regInclude・regExclude）で対象を絞り込みます。

データイメージ例: 配下のサブディレクトリをメタ情報テーブルとして生成（カラムは file と同じ。SIZE_KB は常に 0）。

入力 (src = project/):
  project/src/
  project/src/main/
  project/src/test/

指定:
  srcType   = dir
  recursive = true

テーブル名: project  (ルートディレクトリ名)
カラム:     [PATH, NAME, DIR, RELATIVE_PATH, SIZE_KB, LAST_MODIFIED]
データ行:
  [/abs/project/src,      src,  /abs/project,     src,      0, 2026/04/01 10:00:00]
  [/abs/project/src/main, main, /abs/project/src, src/main, 0, 2026/04/01 10:05:00]
  [/abs/project/src/test, test, /abs/project/src, src/test, 0, 2026/04/02 09:00:00]

none

src の意味: 指定不要（入力を持たない）。

テーブル名: テーブルを生成しません。

固有オプション: なし。

データイメージ例: 入力・生成テーブルがないため、例示すべきデータはありません。テンプレート処理のみ行う Generate などで用います。

Source

入力元となるファイルやクエリ、補助的なオプションを指定します。

項目説明
src入力元のパス。Source Type によって意味が異なる: ファイル・ディレクトリ（csv / fixed / reg / xls / xlsx / file / dir）、SQL ファイル（sql / csvq）、テーブル名リストファイル（table）
encodingテキスト系入力の文字コード
jdbcPropertiesTable / SQL を選んだ場合の JDBC 接続情報（URL・ユーザ・パスワードなど）
templateGroupテンプレート処理を行う場合の補助設定

traversal option

ディレクトリを src に指定した際の走査条件です。traversal option の展開ボタンから設定します。

項目説明
extension対象とするファイル拡張子
recursiveサブディレクトリを再帰的に走査するか
regInclude対象ファイル名の包含条件（正規表現）
regExclude対象ファイル名の除外条件（正規表現）

Data Load

読み込んだデータをどのようにテーブル化するかを指定します。data load option の展開ボタンから設定します。

項目説明
headerNameカラム名をカンマ区切りで明示指定する。指定した場合、先頭行もデータ行として扱われる（詳細は下記）
startRowデータ読み込みを開始する行番号。1 始まり、デフォルトは 1（詳細は下記）
addFileInfoON にすると、元ファイル情報を表す 3 カラム（$FILE_PATH / $FILE_NAME / $SHEET_NAME）を各行に追加する（詳細は下記）
loadData実データを読み込むか（メタデータのみが必要な場合に OFF）
includeMetaDataメタデータカラムを含めるか
regTableInclude処理対象とするテーブル名の正規表現
regTableExclude処理対象から除外するテーブル名の正規表現

headerName の詳細

カラム名をカンマ区切りで与えると、その名前順にテーブル列が構築されます。指定された場合は先頭行も自動的にはヘッダとみなされず、すべての行がデータ行になります。既にヘッダ行が存在するファイルでヘッダを上書きしたい場合は、startRow と併用してヘッダ行をスキップしてください。

例 1: ヘッダ行がない CSV に列名を与える

入力 (users.csv):
John,30
Jane,25

指定:
  headerName = name,age

結果:
  カラム:   [name, age]
  データ行: 2 行（John,30 / Jane,25）

例 2: 既存ヘッダを別名に差し替える

入力 (users.csv):
col_a,col_b
John,30
Jane,25

指定:
  headerName = name,age
  startRow   = 2             (元のヘッダ行をスキップ)

結果:
  カラム:   [name, age]
  データ行: 2 行

startRow の詳細

読み込みを開始する行番号を 1 始まりで指定します。未指定時は 1（ファイル先頭）です。タイトル行やメタ情報行をスキップしたいときに使用します。Excel 入力にも同様に適用されます。headerName を併用しない場合、startRow で指定した行がヘッダ行として扱われ、その次の行以降がデータになります。

例 1: タイトル行をスキップする

入力 (sales.csv):
  1 行目: 売上レポート Q4
  2 行目: id,name
  3 行目: 1,Alice
  4 行目: 2,Bob

指定:
  startRow = 2

結果:
  ヘッダ:   [id, name]   (2 行目)
  データ行: 2 行         (3 行目以降)

例 2: タイトル行＋メタ情報行をまとめてスキップする

入力 (report.csv):
  1 行目: Report
  2 行目: Generated 2026-04-20
  3 行目: id,name
  4 行目: 1,Alice

指定:
  startRow = 3

結果:
  ヘッダ:   [id, name]   (3 行目)
  データ行: 1 行以上     (4 行目以降)

例 3: headerName と併用する

入力 (no_header.csv):
  1 行目: # 事前コメント
  2 行目: John,30
  3 行目: Jane,25

指定:
  startRow   = 2
  headerName = name,age

結果:
  カラム:   [name, age]           (headerName から)
  データ行: 2 行                   (2 行目以降すべてデータ)

addFileInfo の詳細

ON にすると、読み込んだ各行の末尾に元ファイル情報を示す次の 3 カラムが自動で追加されます。

カラム内容
$FILE_PATH入力ファイルの絶対パス（区切りはスラッシュに正規化）
$FILE_NAME入力ファイルのファイル名（例: users.csv）
$SHEET_NAMEExcel の場合はシート名。CSV など単一表の場合は空文字

ディレクトリを src に指定して複数ファイルをまとめて読み込む際、どの行がどのファイル由来かを追跡したい場合に有効です。Compare の差分レポートでソース特定にも利用できます。

例 1: CSV ファイル 1 件を読み込む場合

入力 (data/users.csv):
id,name
1,Alice

指定:
  addFileInfo = ON

結果:
  カラム: [id, name, $FILE_PATH, $FILE_NAME, $SHEET_NAME]
  データ行:
    [1, Alice, /abs/path/data/users.csv, users.csv, ""]

例 2: Excel の複数シートを読み込む場合

入力 (data/book.xlsx):
  Sheet "Users":  id,name / 1,Alice
  Sheet "Orders": id,amount / 10,500

指定:
  addFileInfo = ON

結果 (テーブル "Users" の 1 行):
  [1, Alice, /abs/path/data/book.xlsx, book.xlsx, Users]
結果 (テーブル "Orders" の 1 行):
  [10, 500, /abs/path/data/book.xlsx, book.xlsx, Orders]

比較対象から追加カラムを除外したい場合は、Dataset Settings の除外カラム設定と併用してください。

Setting

Dataset Settings ファイルを指定することで、テーブルごとの主キー・ソート順・フィルタなどのメタデータを与えます。

項目説明
settingDataset Settings ファイルのパス。書式は Dataset Settings を参照
settingEncodingDataset Settings ファイルの文字コード

テキストボックス横の編集ボタンから Dataset Setting ダイアログを開き、テーブルごとの設定を GUI で編集できます。
