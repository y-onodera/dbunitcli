Compare

データセットを比較して差分を検出します。旧データと新データを突き合わせ、差分があれば出力します。

オプション

オプション説明
targetType比較対象の種類（下記参照）
setting比較設定ファイル（data モード時のみ有効。Dataset Settings 形式）
settingEncoding設定ファイルのエンコーディング（デフォルト: システム既定）
oldData比較元（旧）データセット
newData比較対象となる新データセット
expectData期待値データセット（省略時はスキップ）
imageOption画像比較のオプション（image / pdf モード時のみ表示）
convertResult出力形式と出力先の設定

oldData / newData / expectData の入力フォーム詳細は Dataset Load Form を参照してください。

targetType ごとの比較方法

targetType には data・image・pdf の 3 種類があります。

data（デフォルト）

CSV・Excel・DB テーブルなど、表形式のデータセットを比較します。setting に指定した Dataset Settings ファイルで主キー・カラム選択・ソート順などを制御できます。

  • oldData と newData で対応するテーブルを 1 対 1 に突き合わせます
  • テーブルの一致は名前で行われます。名前が対応しないテーブルは追加／削除として検出されます
  • setting を省略するとすべてのカラムを対象に順序比較します

image

PNG 画像ファイルをピクセル単位で比較します。oldData / newData には Source Type = file のディレクトリを指定してください。拡張子は自動的に .png に固定されます。

  • ファイル名が一致するペアを 1 対 1 で突き合わせます
  • 差分の許容度や描画方法は imageOption（Image Option）で調整できます
  • setting は無視されます（比較キーは内部で NAME 固定）

pdf

PDF ファイルをページ画像として比較します。oldData / newData には Source Type = file のディレクトリを指定してください。拡張子は自動的に .pdf に固定されます。

  • ファイル名が一致するペアを 1 対 1 で突き合わせます
  • 差分の許容度や描画方法は imageOption（Image Option）で調整できます
  • setting は無視されます（比較キーは内部で NAME 固定）

setting による比較内容の変更（data モード）

targetType=data のとき、setting に Dataset Settings 形式の JSON ファイルを指定することで、テーブルごとに比較内容を細かく制御できます。

主な制御項目

設定フィールド比較への影響
keys比較の主キーとなるカラムを指定。指定がない場合は行順序で比較
include比較対象に含めるカラムを限定
exclude比較対象から除外するカラム（タイムスタンプ列など差分を無視したい列に使用）
order比較前にデータをソートするカラム
filter比較前に行を絞り込む JeXL 条件式

記述例

{
  "settings": [
    {
      "name": "orders",
      "keys": ["order_id"],
      "exclude": ["created_at", "updated_at"],
      "order": ["order_id"]
    }
  ],
  "commonSettings": [
    {
      "exclude": ["created_at", "updated_at"]
    }
  ]
}

この例では orders テーブルを order_id で突き合わせ、タイムスタンプ列を比較対象から除外します。commonSettings の exclude はすべてのテーブルに適用されます。

filePath を使って別フォルダの同名ファイルに異なる設定を適用する

CSV や Excel を再帰的に読み込む場合、異なるサブフォルダに同名ファイルが存在することがあります。name を { "any": "テーブル名", "filePath": "正規表現" } 形式で指定すると、ファイルパスで絞り込んだうえで設定を適用できます。

たとえば次のようなフォルダ構成で、summary.csv が 2 か所に存在するケースを考えます。

old/
  domestic/
    summary.csv   ← 主キー: region_code
  overseas/
    summary.csv   ← 主キー: country_code

この場合、name にテーブル名と filePath の正規表現を組み合わせることで、フォルダごとに異なる設定を適用できます。

{
  "settings": [
    {
      "name": { "any": "summary", "filePath": ".*/domestic/.*" },
      "keys": ["region_code"],
      "order": ["region_code"]
    },
    {
      "name": { "any": "summary", "filePath": ".*/overseas/.*" },
      "keys": ["country_code"],
      "order": ["country_code"]
    }
  ]
}

filePath の値はファイルパス全体に対する Java 正規表現です。.*/domestic/.* はパス中に domestic/ を含むファイルにのみマッチします。どちらの設定にもマッチしないファイルは commonSettings が適用されます（指定がなければ全カラム順序比較）。

比較結果の形式

比較結果は COMPARE_RESULT という名前のテーブルとして出力されます。convertResult で指定した形式（CSV・Excel 等）でファイルに書き出されます。

COMPARE_RESULT テーブルのカラム

カラム説明
targetName差分が検出されたテーブル名（image/pdf モードではファイル名）
diff差分の種類（下表参照）
oldDefine旧データ側の値・定義
newDefine新データ側の値・定義
columnIndex差分が発生したカラムの位置（0 始まり）
rows差分が発生した行番号（0 始まり）

diff の値一覧

値意味
TABLE_COUNTテーブル数が異なる
TABLE_DELETE旧データにあるテーブルが新データに存在しない
TABLE_ADD新データに追加されたテーブルが旧データに存在しない
COLUMNS_COUNTカラム数が異なる
COLUMNS_MODIFY同位置のカラム名が異なる
COLUMNS_DELETE旧データにあるカラムが新データに存在しない
COLUMNS_ADD新データに追加されたカラムが旧データに存在しない
ROWS_COUNT行数が異なる
KEY_DELETE旧データにある主キー行が新データに存在しない
KEY_ADD新データに追加された主キー行が旧データに存在しない
MODIFY_VALUE同一キーの行でカラム値が異なる

expectData を指定した場合

expectData に期待値データセットを指定すると、convertResult の出力結果と期待値を再度突き合わせて検証します。期待値と一致しない差分は convertResult 配下の expectedDiff/ ディレクトリに同形式で出力されます。
